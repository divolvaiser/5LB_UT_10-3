Обновление правил обмена с конфигурацией "Бухгалтерия предприятия"

Обработка используется в контексте обновления с версии 10.3.4.14 на 10.3.5

В связи с тем, что изменилась структура хранения данных о настройках обмена необходимо внести изменения в информационную базу, что бы сохранить работоспособность механизма автоматического обмена данными.

 

В версии 10.3.4.14 и ниже в справочнике “Настройки обмена данными” хранилась информация связанная непосредственно с тем, куда и как нужно выгружать и загружать данные и с тем когда это нужно делать. В версии 10.3.5 структура хранения информации меняется. Добавляется новый справочник “Настройки выполнения обмена” в котором хранится информация об автоматическом обмене данными и о настройках, по которым этот обмен необходимо выполнить. Из справочника “Настройки обмена данными” информация об автоматическом обмене данными удаляется.

 

Что позволяет выполнять такой подход к настройкам обмена:

Возможность настройки группового последовательного выполнения обмена данными. То есть можно создать настройки для каждого узла обмена центральной информационной базы и одной настройкой автоматического выполнения обмена указать, что нужно провести последовательную загрузку данных для каждого узла, а затем последовательную выгрузку.

Возможность задавать несколько различных расписаний выполнения обмена данными для одной настройки.

 

Обработка находит настройки обмена данными, для которых были выполнены настройки автоматического обмена и для каждой такой настройки создает отдельный элемент в справочнике “Настройки выполнения обмена”. После этого обработка УДАЛЯЕТ все регламентные задания, которые относились к старой системе настроек автоматического выполнения обменов.

 

Рекомендуется после выполнения обработки проверить правильность и корректность создания настроек автоматического выполнения обменов данными.

 

Обработку допускается выполнять только ОДИН раз. В случае повторного запуска возможно задвоение элементов справочника "Настройки выполнения обмена". В том случае если после первого запуска обработки есть настройки обмена, по которым не созданы настройки автоматического выполнения и они необходимы, то рекомендуется создать их в ручном режиме.
