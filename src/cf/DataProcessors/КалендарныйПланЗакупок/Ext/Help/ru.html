Обработка «Календарный план закупок»

Данная обработка является своеобразным АРМ менеджера по закупкам и предназначена для формирования заказов поставщикам на основании потребностей в номенклатуре.

После формирования потребностей в номенклатуре менеджер по закупкам может воспользоваться возможностями данной обработки.

Для текущего пользователя определяются те номенклатурные позиции, по которым он является ответственным за закупки, и только эти товары участвуют в формировании заказов поставщикам.

Если существуют неудовлетворенные потребности, то при открытии обработки автоматически открывается помощник заполнения календарного плана закупок, в противном случае сразу открывается основная форма календарного плана закупок.

Помощник заполнения календарного плана закупок

В предложенной таблице выбираются те номенклатурные позиции, по которым необходимо подбирать поставщиков и формировать по ним заказы.

Переходя по кнопке «Далее», пользователь может настроить различные отборы, по которым будут подбираться поставщики для номенклатурных позиций.

По кнопке «Далее» производится определение поставщиков, которые могут поставлять выбранные товары, на основании данных регистра «Номенклатура контрагентов», определяются цены контрагентов на каждую номенклатурную позицию, для каждой цены указываются тип цен или особые условия поставки по договору с контрагентом.

Пользователь флажками может отметить, какие цены необходимо подставлять в формируемые договора.

Цвет текста в ячейках определяет следующее:

  • Темно-красный, данная цена является минимальной из всех зарегистрированных цен поставщиков и данный поставщик указан как основной поставщик товара
  • Зеленый, данная цена является минимальной из всех зарегистрированных цен поставщиков
  • Синий, данный поставщик указан как основной поставщик товара

В контекстном меню таблицы можно воспользоваться сервисными функциями, которые помогают принять решение о выборе поставщика.

Все цены в табличной части отображаются в валюте управленческого учета, для возможности их сравнения.

При нажатии на кнопку «Готово» по указанным номенклатурным позициям формируются заказы выбранным поставщикам.

Открывается основная форма календарного плана закупок.

Основная форма календарного плана закупок

Основная форма календарного плана закупок представлена в виде дерева, которое может отображаться в 2-х режимах:

  • Номенклатура - Дата потребности - Контрагент - Заказ поставщику
  • Контрагент - Номенклатура - Дата потребности - Заказ поставщику

Номенклатура и дата потребности определяются на текущий момент по данным проведения документов «Формирование потребностей». Можно указать анализируемый период дат потребности регистра «Календарные потребности в номенклатуре».

При заполнении дерева анализируются заказы поставщикам которые еще не закрыты, т.е. по которым имеются не поставленные до конца номенклатурные позиции в разрезе характеристик номенклатуры и признака Товар/Тара.

Красным цветом отмечены строки, по которым потребности не удовлетворены в полном объеме, синим - по которым количество заказанных товаров превышает необходимые потребности.

Заказ поставщику может находиться в 3-х состояниях, которые отмечаются соответствующими пиктограммами:

  • Заказ записан в ИБ, проведен и не был изменен в текущем сеансе работы календарного плана закупок (без пиктограммы)
  • Заказ записан в ИБ, проведен и был изменен в текущем сеансе работы календарного плана закупок
  • Заказ еще не записан в ИБ, а был создан в текущем сеансе работы календарного плана закупок

Пользователь может создать заказ без использования помощника заполнения. Для этого необходимо выделить несколько строк дерева значений, которые проанализируются программой, и добавить новую строку в дерево. Откроется форма нового заказа поставщику, в которую автоматически подставится минимальная дата потребности из выбранных (как дата поставки), контрагент (как поставщик) если он присутствовал в выбранных строках дерева и заполниться табличные части номенклатуры и возвратной тары документа. В форме документа пользователь должен указать только цены номенклатуры.

ВАЖНО:

Все заказы поставщикам, созданные или измененные в течение сеанса работы с календарным планом закупок, записываются в ИБ только по нажатии кнопки «Выполнить» нижней командной панели основной формы календарного плана закупок. В процессе работы с ними в календарном плане закупок - данные о заказах хранятся динамически, в памяти.

Если заказ под какую-либо номенклатурную позицию сформирован, но потребности не удовлетворены до конца, то при нажатии кнопки «Автокорректировка заказа» командной панели дерева значений, откроется форма заказа с откорректированными под потребности значениями «количество» для каждой номенклатурной позиции, для которой это требуется для удовлетворения потребностей.

Наряду со стандартными отборами дерева значений (по значениям каждого поля дерева) можно воспользоваться дополнительными отборами:

  • Отбирать только не заказанные товары (количество заказанных товаров меньше чем количество потребности)
  • Отбирать не выполненные даты потребности
  • Отбирать даты поставки товаров, не заявленные в плане потребности
  • Возможность отбирать заказы по их состоянию

В режиме отображения дерева Контрагент - Номенклатура - Дата потребности - Заказ суммирование значений по некоторым полям дерева значений - не имеет смысла, поэтому те поля и значения, которые можно суммировать, выделены другим цветом (цвет формы).

Для оперативной установки разворота строк дерева можно воспользоваться полем выбора уровня разворота строк дерева значений.

Помощник установки соответствий заказов покупателей и заказов поставщикам.

В заказах поставщикам можно указывать размещение, т.е. тот заказ покупателя, под который конкретно заказывается товар поставщику. Пользователь может открывать форму каждого заказа поставщику и указывать размещение вручную, а может воспользоваться помощником заполнения соответствий заказов покупателей и заказов поставщикам.

По кнопке в верхней командной панели формы, открывается форма помощника. Данные так же представлены в виде дерева в порядке Номенклатура - Дата потребности - Заказ покупателя - Заказ поставщику. При открытии формы помощника программа автоматически распределяет заказы, и в табличные части заказов поставщикам вносит корректировки - проставляет заказы покупателей для которых производится заказ товара поставщику с учетом дат отгрузки (заказ покупателя) и дат поставки товаров (заказ поставщику), при необходимости разбивает одну строку на несколько.

Пользователь может откорректировать заказы поставщикам, заказы покупателей для корректировки недоступны.

Все изменения, сделанные данным помощником, переносятся в дерево основной формы календарного плана закупок только при нажатии кнопки «Выполнить».
