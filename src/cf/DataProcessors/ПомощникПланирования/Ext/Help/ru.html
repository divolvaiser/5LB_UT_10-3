Обработка "Помощник планирования"

Целью данной обработки является помощь в формировании документов планирования по данным подсистемы планирования и смежных подсистем.

Данная обработка может быть вызвана из главного меню или из форм документов "План закупок" и "План продаж" по нажатию кнопки "Заполнить" командной панели табличного поля "Номенклатура". В случае вызова обработки из форм документов внешний вид обработки имеет сокращенный вид.

Для формирования документов необходимо определить конечные планы. Данная информация указывается в табличном поле на странице "Конечные планы". Конечные планы подразделяются по виду (закупки и продажи), сценарию (версии), периоду действия.

Конечные планы.

  • Вид плана - определяет, какого вида план необходимо сформировать (закупки или продажи).
  • Сценарий - сценарий конечного плана (версия), определяющий периодичность, детализацию планирования (номенклатура или номенклатурная группа) и состав показателей (суммовой и/или количественный учет).
  • Дата нач. - определяет начальную дату интервала формируемых планов.
  • Дата кон. - определяет конечную дату интервала формируемых планов.
  • Проект - определяет дополнительный аналитический разрез, который будет присвоен сформированному плану (необязательный).
  • Подразделение - определяет дополнительный аналитический разрез, который будет присвоен сформированному плану (необязательный).
  • Профиль распределения - профиль, по которому будет выполняться распределение (необязательный).
Примечание.

Выбор дат начала и окончания периода осуществляется раздельно. Для удобства выбора дат в форме настройки периода необходимо в командной панели табличного поля включить выбор дат периодом (кнопка панели "Выбирать даты периодом").

Интервал, определяемый датами начала и окончания, указывается кратным периодичности сценария. В противном случае границы интервала будут выровнены автоматически. Например, при указании сценария с периодичностью "месяц" и интервалом "05.01.2004 - 20.01.2004" после выравнивания будет получен интервал "01.01.2004 - 31.01.2004".

Возможно указание интервала, превышающего по длительности период сценария. Например, при указании периода "05.01.2004-20.03.2004" будет произведено выравнивание и будет получен интервал "01.01.2004-31.03.2004". Важным моментом является то, что в этом случае будет сформировано несколько планов, количество которых будет равно количеству вхождений периода сценария в указанный период. Для приведенного примера будет сформировано 3 документа.

В случае вызова обработки из форм документов "План закупок" и "План продаж" данные по конечным планам не заполняются и данная страница панели отключается. Параметры конечного плана в данном случае определяются параметрами заполняемого документа.

После определения конечных планов необходимо определить источники данных, определяющие состав планов и плановые показатели. Для этого необходимо задать стратегии расчета количества и суммы. Состав и количественные показатели планов определяются по стратегии расчета количества и задаются на странице "Стратегия расчета количества".

Стратегия расчета количества.

Источниками данных для формирования состава и количественных показателей планов могут являться:

Закупки.

В качестве закупок могут быть использованы фактические объемы закупок и планируемые закупки. Обязательным является указание периода выборки данных. Выборка производится за весь указанный период и в соответствии с указанными отборами на странице "Отборы".

Продажи.

В качестве продаж могут быть использованы фактические объемы продаж и планируемые продажи. Обязательным является указание периода выборки данных. Выборка производится за весь указанный период и в соответствии с указанными отборами на странице "Отборы".

Заказы.

В качестве заказов могут быть использованы заказы поставщикам, заказы покупателей и внутренние заказы. Обязательным является указание периода выборки данных. Выборка данных производится по неисполненным частям заказов на дату окончания указанного периода и с учетом даты отгрузки (даты поступления), находящейся в указанном периоде и в соответствии с указанными отборами на странице "Отборы".

Остатки.

В качестве остатков могут быть использованы складские и плановые остатки. Обязательным является указание даты выборки складских остатков и сценария плановых остатков. Выборка данных производится в соответствии с указанными отборами на странице "Отборы".
 
Примечание.

При включении нескольких стратегий, производится суммирование (за исключением описанных ниже особенностей) выбираемых показателей.

Для каждой из включенных стратегий может быть указан процент, определяющий корректировку выбранных количественных показателей.

Для каждой из стратегий возможно указание исключения значений (флаг "Исключить"). Если для стратегии указан признак исключения, то показатели количества считаются отрицательными и для них производится не суммирование, а вычитание. Например, если указаны стратегии расчета количества "Планы продаж" и "Заказы покупателей" и для заказов покупателей указано исключение, то конечные данные будут содержать состав и показатели планов продаж, уменьшенные на показатели заказов покупателей.

При включении стратегии расчет количества "Плановые остатки" выборка производится только при выбранных стратегиях "Планы закупок" или "Планы продаж". Только данные планов могут быть увеличены или уменьшены на размер плановых остатков. Для каждой позиции, выбранной из плановых данных, производится расчет планового остатка на начало периода действия плана и на конец. Разница является величиной изменения плановых остатков для текущей позиции в периоде планирования и именно на эту величину производится корректировка.

Если среди выбираемых данных определены плановые данные, то для них может быть выполнено преобразование, включающее различны варианты сложения или объединения. Важно отметить, что сложение и объединение выполняется только для плановых данных. При сложении плановых данных выполняется группировка по номенклатуре и характеристике номенклатуры и сложение показателей. Включение в группировку подразделения и/или проекта осуществляется выбором соответствующей стратегии ("Сложение (Проект)", "Сложение (Подразделение)", "Сложение (Проект, Подразделение)"). В этом случае при группировке будут учитываться значения проекта и/или подразделения плановых данных. При объединении плановых данных выполняется выбор максимального показателя количества. Поведение при выборе других стратегий объединения полностью соответствует описанному выше для сложения.

По окончании выборки и группировки выбранных данных может быть произведено изменение результат на процент и округление по указанному правилу (раздел "Дополнительно"). Корректировка производится только в случае включения флага "Изменить результат расчета".

Для расчета суммовых показателей необходимо определить стратегии расчета суммы для каждой из выбранных ранее стратегий расчета количества. Стратегия расчета суммовых показателей задается на странице "Стратегия расчета суммы". Важно отметить, что способ расчета суммы действует только для соответствующей стратегии расчета количества и влияет на расчет суммовых показателей только по этой стратегии.

Стратегия расчет суммы.

Стратегия расчета суммы определяет, каким образом необходимо рассчитать суммовые показатели плана. Возможны три способа расчета:

По стратегии расчета количества.

В этом случае расчет суммы выполняться не будет, и данные о суммовых показателях будут получены из источников получения количественных показателей. Например, для стратегии расчета количества "Объемы закупок" данные о суммовых показателях будет получены из фактических продаж. Аналогично и для других стратегий.

По типу цен номенклатуры.

В этом случае будет производиться расчет суммовых показателей на основании цен номенклатуры. Если тип цен номенклатуры указан, то именно по этому типу цен и будут рассчитаны суммовые показатели. В случае не указания типа цен будет производиться расчет сумм по всем существующим типам цен с учетом указанного способа расчета ("Максимум", "Среднее" или "Минимум").

По типу цен контрагентов.

В этом случае будет производиться расчет суммовых показателей на основании цен номенклатуры контрагентов. В качестве типа цены может быть указан тип цен номенклатуры контрагентов или тип цен номенклатуры.
В первом случае, если тип цен номенклатуры контрагентов указан, то именно по этому типу цен и будут рассчитаны суммовые показатели.
Во втором случае, если тип цен номенклатуры указан, то расчет цен будет производиться среди тех типов цен номенклатуры контрагентов, для которых в качестве типа цены номенклатуры указан выбранный тип цены номенклатуры. В этом случае будет производиться расчет цен с учетом указанного способа расчета ("Максимум", "Среднее" или "Минимум").
В случае не указания типа цен будет производиться расчет сумм по всем существующим типам цен с учетом указанного способа расчета ("Максимум", "Среднее" или "Минимум").
 
Примечание.

В случае указания способов расчета "По типу цен номенклатуры" или "По типу цен контрагентов" обязательным является указание даты расчета цены.

Рассчитанные суммы могут быть изменены на процент и округлены по указанному правилу (раздел "Дополнительно"). Корректировка производится только в случае включения флага "Изменить результат расчета".

Отборы.

Состав доступных полей отбора разделен по видам стратегии расчета количества. Таким образом, для установки отбора по какому либо показателю необходимо включение отбора по каждой из выбранных стратегий. Состав доступных отборов изменяется динамически и зависит от выбранных стратегий расчета суммы. Для установки отбора по свойствам и категориям объектов необходимо включить флаг "Использовать свойства и категории" в нижней части страницы "Отборы".

По окончании заполнения показателей необходимо выполнить формирование нажатием кнопки "Выполнить" нижней командной панели формы. В случае вызова обработки из формы документов "План закупок" и "План продаж" и успешного выбора и расчета показателей будет заполнена табличная часть соответствующего документа. В случае независимого вызова обработки из главного меню будут и успешного выбора и расчета показателей будут сформированы и записаны документы. Сформированные документы будут помещены в табличное поле на странице "Сформированные документы".

Сформированные документы.

После успешного формирования документов табличное поле будет содержать документы, которые можно просматривать, редактировать и выполнять другие операции, доступные в командной панели табличного поля.

Примечание.
В случае вызова обработки из форм документов "План закупок" и "План продаж" данная страница панели отключается. Сформированным документом является заполняемый документ.

Сохранение настроек.

Обработка позволяет сохранять все настройки для последующего восстановления. Режим сохранения при вызове из документов "План закупок" и "План продаж" отличается от режима сохранение при вызове из главного меню. При вызове из главного меню восстановление и сохранение настроек производится стандартным способом с использование кнопок верхней панели формы "Восстановить значения" и "Сохранить значения". При вызове из документов "План закупок" и "План продаж" сохранение настроек производится автоматически для каждого документа только в случае успешного заполнения. При последующем вызове из сохраненного документа настройки восстанавливаются.
