Обработка "Выгрузка данных в 1С:Предприниматель 7.7"

Обмен данными между конфигурациями "Управление торговлей" и "1С:Предприятие 7.7, Учет и отчетность предпринимателя" осуществляется на уровне документов.
Обмен данными с конфигурацией "1С:Предприятие 7.7, Учет и отчетность предпринимателя" производится для всех редакций, имеющих номер не ниже 1.2.
В конфигурацию "1С:Предприятие 7.7, Учет и отчетность предпринимателя" выгружаются только те документы, для которых установлен признак "Отражать в регл.учете".

Для осуществления выгрузки данных следует в меню "Сервис" выбрать пункт меню "Выгрузка данных в 1С:Предприниматель 7.7".

Внимание. Выгрузку данных в конфигурацию "1С:Предприятие 7.7, Учет и отчетность предпринимателя" рекомендуется осуществлять после проведения регламентного документа "Проведение по партиям".

Выгрузка осуществляется по тому юридическому лицу, который выбирается в настройках выгрузки. Отбираются те документы, для которых  юридическое лицо установлено в качестве юридического лица организации, выбранной в документе.  

Перед выгрузкой данных можно осуществить отбор документов.
Установка значений отбора производится аналогично установке значений отбора в настройках отчетов. Так же, как для отчетов можно использовать для отбора одиночный и множественный отборы.

Отбор документов можно произвести по реквизитам: склад и подразделение.

Перед выгрузкой документов необходимо загрузить правила конвертации объектов. Для этого на закладке "Правила выгрузки данных" надо нажать кнопку "Загрузить правила обмена".

После загрузки правил можно снять флаги у тех документов, которые не нужно выгружать.

В настройках выгрузки необходимо установить период, за который выгружаются документы.

Для учета продаж, облагаемых единым налогом на вмененный доход (ЕНВД) при выгрузке документов можно установить флаг "НДС включать в себестоимость" . 

Выгрузку документов можно производить поэтапно. Например, сначала можно отобрать документы тех складов, с которых продажи производятся без учета ЕНВД (единого налога на вмененный доход). При этом  флаг "НДС включать в себестоимость" не устанавливать и выгрузить документы. Затем отобрать для выгрузки документы тех складов, продажи которых облагаются ЕНВД, установить флаги "НДС включать в себестоимость", "Розница облагается ЕНВД" и произвести выгрузку очередной группы документов.

Важно. При осуществлении выгрузки из 1С: Управление торговлей 8, ред. 10.3 в 1С:Предприниматель 7.7 не выгружаются документы "Требования-накладные". При осуществлении выгрузки из 1С: Управление торговлей 8, ред. 10.3 в 1С:Предприниматель 7.7, предполагается, что производственный учет ведется в 1С:Предпринимателе 7.7.  Поэтому документы "Требования-накладные" в 1С:Предприниматель 7.7 не выгружаются.

Вместе с выгрузкой документов производится выгрузка элементов справочников, которые используются в документах.

При выгрузке операций поступления и отгрузки комиссионных товаров в конфигурации "1С:Предприятие 7.7, Учет и отчетность предпринимателя" будут созданы документы, по вспомогательному виду деятельности "Прием на реализацию (служебный)".

Информация о возвратной таре в конфигурацию "1С:Предприятие 7.7, Учет и отчетность предпринимателя" не выгружается.

При осуществлении выгрузки документов можно установить флаги "Не замещать справочники при загрузке" и "Не замещать документы при загрузке". В этом случае данные, выгруженные ранее, при повторной загрузке не будут замещены данными последующей загрузки. Если флаги не установлены, то данные замещаются при последующей загрузке.
Идентификация записей справочников производится по коду, а идентификация документа - по виду документа, его номеру и дате.

Внмание. Длина поля реквизита "Наименование" в справочнике "Контрагенты" в конфигурации "Управление торговлей" больше, чем длина аналогичного поля в конфигурации "1С:Предприятие 7.7, Учет и отчетность предпринимателя", поэтому при загрузке в 1С:Предприниматель длинные наименования контрагентов могут быть загружены неполностью.

Выгрузка данных производится во внешний файл в формате .xml.

При загрузке данных в конфигурации "1С: Предприятие 7.7" следует пользоваться универсальной загрузкой данных.

Документы в конфигурацию "1С:Предприятие 7.7, Учет и отчетность предпринимателя" загружаются непроведенными. Для проведения группы документов следует воспользоваться обработкой "Обработка документов" конфигурации "Учет и отчетность предпринимателя".
