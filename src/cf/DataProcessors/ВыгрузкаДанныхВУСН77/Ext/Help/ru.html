Обработка Выгрузка данных в конфигурацию "Упрощенная система налогообложения"

Обмен данными между конфигурациями "Управление торговлей" и "1С:Предприятие 7.7, Упрощенная система налогообложения" осуществляется на уровне документов.
Обмен данными с конфигурацией "1С:Предприятие 7.7,  Упрощенная система налогообложения" производится для всех редакций, имеющих номер не ниже 1.34.
В конфигурацию "1С:Предприятие 7.7, Упрощенная система налогообложения" выгружаются только проведенные документы, для которых установлен признак "Отражать в бух.учете".

Для осуществления выгрузки данных следует в меню "Сервис/Обмен данными" выбрать пункт меню "Выгрузка данных в конфигурацию "Упрощенная система налогообложения"".

Внимание. Выгрузку данных в конфигурацию "1С:Предприятие 7.7, Упрощенная система налогообложения" следует осуществлять после проведения регламентной обработки "Проведение по партиям".

Выгрузка осуществляется по той организации, которая выбирается в настройках выгрузки.   

Перед выгрузкой данных можно осуществить отбор документов.
Установка значений отбора производится аналогично установке значений отбора в настройках отчетов. Так же, как для отчетов можно использовать для отбора одиночный и множественный отборы.

Отбор документов можно произвести по реквизиту "склад".

При отборе документов по складам, отбор производится по тому складу, который указан в шапке документа. Склады, указанные в табличной части документа в отборе не участвуют.

Перед выгрузкой документов необходимо загрузить правила конвертации объектов. Для этого на закладке "Правила выгрузки данных" надо нажать кнопку "Загрузить правила обмена".

После загрузки правил можно снять флаги у тех документов, которые не нужно выгружать.

В настройках выгрузки необходимо установить период, за который выгружаются документы и выбрать объект налогообложения при применении УСН.

Выгрузку документов можно производить поэтапно. Например, по разным складам.

Вместе с выгрузкой документов производится выгрузка элементов справочников, которые используются в документах.

При выгрузке данных, относящихся к поступлению и отгрузке комиссионных товаров, в справочник "Номенклатура" в конфигурации "1С:Предприятие 7.7, Упрощенная система налогообложения" будет добавлена запись о позиции номенклатуры с типом номенклатуры "Товар на комиссии". Если один и тот же товар был куплен как собственный и комиссионный товар, то в справочнике "Номенклатура" будет две записи по одному и тому же товару: собственный товар и товар на комиссии.

Документ "Инвентаризация товаров" выгружается с установленным флагом "Формировать проводки". Поэтому после проведения данного документа в конфигурации "1С:Предприятие 7.7, Упрощенная система налогообложения" будут автоматически сформированы проводки по списанию и оприходованию товаров. Документы "Списание товаров" и "Оприходование товаров" в конфигурацию "1С:Предприятие 7.7, Упрощенная система налогообложения" не выгружаются.

Документы "Инвентаризация товаров" оформленные на инвентаризацию НТТ не выгружаются. В этих случаях, списание товара производится документом "Отчет о розничных продажах" Этот документ будет загружен в "1С:Предприятие 7.7, Упрощенная система налогообложения" в качестве документа "Реализация" с типом отгрузки "Продажа в розницу".

Информация о возвратной таре в конфигурацию "1С:Предприятие 7.7, Упрощенная система налогообложения" не выгружается.

При осуществлении выгрузки документов можно установить флаги "Не замещать справочники при загрузке" и "Не замещать документы при загрузке".

В этом случае данные, выгруженные ранее, при повторной загрузке не будут замещены данными последующей загрузки. Если флаги не установлены, то данные замещаются при последующей загрузке.
Идентификация записей справочников производится по коду, а идентификация документа - по виду документа, его номеру и дате.

Внмание. Длина поля реквизита "Наименование" в справочнике "Контрагенты" в конфигурации "Управление торговлей" больше, чем длина аналогичного поля в конфигурации "1С:Предприятие 7.7, Упрощенная система налогообложения", поэтому при загрузке в бухгалтерию длинные наименования контрагентов могут быть загружены неполностью.

Выгрузка данных производится во внешний файл в формате .xml.

При загрузке данных в конфигурации "1С: Предприятие 7.7" следует пользоваться универсальной загрузкой данных.

Документы в конфигурацию "1С:Предприятие 7.7, Упрощенная система налогообложения" загружаются непроведенными. Для проведения группы документов следует воспользоваться обработкой "Обработка документов" конфигурации "Упрощенная система налогообложения".
